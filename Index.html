Introduction to Git and Github
What is Git?

Git is a distributed version control system designed to track changes in source code during software development. It helps teams collaborate by allowing multiple developers to work on the same project simultaneously, without overwriting each other's changes. Git also keeps a history of all changes, enabling developers to revert to previous versions of their code if necessary.

What is GitHub?

GitHub is a cloud-based hosting service for Git repositories. It provides a web-based graphical interface along with features like issue tracking, pull requests, and wikis. GitHub is widely used for project collaboration, allowing developers to contribute to open-source projects, manage repositories, and share their work with others.

Setting up Git
Installing Git

To install Git on your computer, follow these steps:

  • Windows: Download Git from git-scm.com and follow the installation instructions.
  • Mac: Use Homebrew to install Git by running the following command in the terminal: brew install git
  • Linux: Install Git using the package manager for your distribution: sudo apt-get install git # Ubuntu/Debian sudo yum install git # Fedora

Configuring Git

After installing Git, you need to configure it with your user information, which will be associated with your commits.

Set your username:

git config --global user.name "Your Name"

Set your email:

git config --global user.email "your.email@example.com"
Basic Git Workflow
Initializing a Git Repository

To start tracking changes in a project, navigate to the project's directory and run the following command to initialize a Git repository:

git init

This creates a .git directory, where Git stores all the version control information.

Adding Files to Git

To stage files for commit (i.e., to prepare them to be saved to the repository), use:

git add file.txt

To add all files in the current directory, use:

git add .
Commiting Changes

Once files are staged, commit them to the repository with:

git commit -m "Your commit message"

The -m flag allows you to write a short message describing the changes.

Branching and Merging
Creating a Branch

Branching in Git allows you to work on different versions of a project simultaneously. To create a new branch, use the command:

git branch new-feature

This creates a new branch called new-feature. You can then switch to this branch using:

git checkout new-feature
Merging Branches

Once you've made changes in a branch, you can merge them back into the main branch (typically master or main) with:

git checkout main
git merge new-feature

This merges the new-feature branch into the main branch.

Working with Remote Repositories
Adding a Remote Repository

A remote repository allows you to collaborate with others by sharing your code online. To add a remote repository, use:

git remote add origin https://github.com/username/repository.git

Now you can push your changes to this remote repository.

Pushing Changes to Remote

To push your commits to the remote repository, use:

git push origin main

This will push the changes from your local main branch to the origin remote repository.

Cloning Repositories

To work on an existing GitHub repository, you can clone it to your local machine. Use the following command:

git clone https://github.com/username/repository.git

This will download a copy of the repository to your computer, allowing you to make changes locally.

Collaborating on GitHub
Forking a Repository

To contribute to an open-source project, you typically fork the repository to create your own copy. You can do this directly from the GitHub interface by clicking the "Fork" button.

Making a Pull Request

After forking and making changes to your own copy, you can create a pull request to propose these changes to the original project. On GitHub, go to the original repository and click "New pull request".

Resolving Merge Conflicts

Sometimes, Git will encounter conflicts when merging branches. This happens when the same part of a file is modified in two different branches. Git will mark the conflict in the file, and you'll need to edit the file manually to resolve the conflict.

After resolving the conflict, mark it as resolved by adding the file again:

git add conflicted-file.txt

Then commit the changes:

git commit -m "Resolved merge conflict"
Common Git Commands Cheat Sheet

Here are some of the most commonly used Git commands:

  • Check the status of your files: git status
  • View the commit history: git log
  • See changes made to files: git diff
  • Temporarily save changes without committing: git stash
  • Undo the last commit (while keeping changes): git reset --soft HEAD~1
